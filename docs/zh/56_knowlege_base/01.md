# kb 主要数据库设计

本章节旨在引导大家从零开始实现一个基于 RAG（Retrieval-Augmented Generation）的知识库项目。

## 数据选型

- **前端**：使用 [MaxKB 前端](https://github.com/1Panel-dev/MaxKB)
- **后端**：采用 Java 进行开发
- **模型端**：使用 OpenAI-4o-mini 系列模型
- **数据端**：使用 PostgreSQL 配合 pgvector 进行向量数据的存储

## 数据库设计

### 规范

- 主键:主键统一使用雪花 id

### 表结构解释

1. **`max_kb_user`**

   存储用户的信息，包括用户 ID、邮箱、电话、昵称、用户名、密码、角色、激活状态、来源、备注、创建者、创建时间、更新者、更新时间、删除标志及租户 ID。

2. **`max_kb_model`**
   关联模型数据
3. **`max_kb_application`**

   存储知识库应用程序的信息，包括应用 ID、代码、名称、描述、课程名称、所有者名称、提示信息、前言、对话数量、数据集设置、模型设置、问题优化选项、模型 ID、用户 ID、图标、类型、工作流程、模型参数设置、状态、备注、创建者、创建时间、更新者、更新时间、删除标志及租户 ID。

4. **`max_kb_user_application`**

   关联用户与应用程序的关系表，每条记录表示一个用户与一个应用程序的关联，包含关联 ID、应用 ID、用户 ID、备注、创建者、创建时间、更新者、更新时间、删除标志及租户 ID。

5. **`max_kb_task`**

   存储任务的信息，包括任务 ID、文件 ID、文件名称、文件大小、数据集 ID、文档 ID、进度、状态、创建者、创建时间、更新者、更新时间、删除标志及租户 ID。

6. **`max_kb_file`**

   存储文件的信息，包括文件 ID、MD5 值、文件名、文件大小、用户 ID、上传平台、区域名、存储桶名称、文件存储 ID、文件存储路径、文件标签（JSON 格式）、创建者、创建时间、更新者、更新时间、删除标志及租户 ID。

7. **`max_kb_dataset`**

   存储数据集的信息，包括数据集 ID、名称、描述、类型、元数据、用户 ID、备注、创建者、创建时间、更新者、更新时间、删除标志及租户 ID。

8. **`max_kb_application_dataset_mapping`**

   关联应用程序与数据集的映射表，表示哪些数据集被哪个应用程序使用，包含映射 ID、应用 ID 及数据集 ID。

9. **`max_kb_document`**

   存储文档的信息，包括文档 ID、标题、名称、字符长度、状态、激活状态、类型、元数据、所属数据集 ID、命中处理方法、直接返回相似度、文件信息（JSON 格式）、创建者、创建时间、更新者、更新时间、删除标志及租户 ID。

10. **`max_kb_paragraph`**
    存储段落的信息，包括段落 ID、来源 ID、来源类型、内容、标题、状态、点击数、激活状态、向量表示（Embedding）、元数据、所属数据集 ID、文档 ID、段落 ID、搜索向量、创建者、创建时间、更新者、更新时间、删除标志及租户 ID。
