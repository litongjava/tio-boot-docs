# AI 问答

## AI 问答

Ai 问答非常容易实现,直接使用笔者的 java-openai 库 调用 openai 的 chat 接口将问题问题发送的 openai,openai 返回推理结果

## 临时记忆

临时记忆的实现有很多种方式,这里简单介绍两种方式

1. 将用户和大模型的对话记录存入到数据库中,需要对新的问题进行推理时 向新的消息和历史消息一起发给大模型.缺点是每一次推理都会增加 token 数量
2. 编写一个提示词,将用户的最新消息和历史记录放到提示词中. 通过这个提示词大模型会将新的消息和历史消息汇总成 一个最新的问题.再将这个问题交给大模型推理.这里可以减少推理的 token 数量,但是会增加一层模型调用

## 长期记忆

长期记忆的实现比较复杂,这里介绍一种简单方式

1. 将用户的 输入 和大模型的 输出 添加到历史表中, 并进行向量
2. 当用户有新的输入时.先对输入进行向量,得到输入向量.将输入向量和记录中的向量进行余弦相似度计算,得到相似度较高的 历史输入和输出
3. 将 相似度较高的 历史输入和输出 和 最新的数据交给大模型推理 或者先 让大模型汇总成 一个输入 在让大模型进行推理

这里做的好处有下面几个优点

1. 减少 token 数量: 并不是 历史输入和输出 中的所有信息都和 最新的输入有关联 通过余弦相似度找出 相似度较高的片段,可以减少 token 梳理
2. 理论上支持无线长的对话: 英文可以通过工程技术找出相似度最高的前 10 条或者前 100 条.理论上永远不会超出 token 限制
