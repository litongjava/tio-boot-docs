# 日志

`tio-boot` 并未与任何特定日志框架绑定，使用的是 `slf4j` 作为日志门面。

## logback

笔者建议在开发 `tio-boot` 应用时使用 `logback` 作为日志实现，以下是其使用方法：

### 1. 引入 logback 依赖

在项目的 `pom.xml` 文件中添加 `logback` 依赖：

```xml
<dependency>
  <groupId>ch.qos.logback</groupId>
  <artifactId>logback-classic</artifactId>
  <version>1.2.3</version>
</dependency>
```

### 2. 配置 logback.xml

在项目的 `resources` 目录下创建 `logback.xml` 配置文件，内容如下：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration debug="false" xmlns="http://ch.qos.logback/xml/ns/logback"
               xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
               xsi:schemaLocation="http://ch.qos.logback/xml/ns/logback https://raw.githubusercontent.com/enricopulatzo/logback-XSD/master/src/main/xsd/logback.xsd">

    <!-- 日志文件存储位置 -->
    <property name="LOG_HOME" value="logs" />

    <!-- 日志格式配置 -->
    <property name="CONSOLE_LOG_PATTERN" value="%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-6level%logger{1}.%M:%L - %m%n" />

    <!-- 控制台输出配置 -->
    <appender name="STDOUT" class="ch.qos.logback.core.ConsoleAppender">
        <encoder class="ch.qos.logback.classic.encoder.PatternLayoutEncoder">
            <pattern>${CONSOLE_LOG_PATTERN}</pattern>
        </encoder>
    </appender>

    <!-- 文件输出配置：按照每小时生成新日志文件 -->
    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <encoder class="ch.qos.logback.classic.encoder.PatternLayoutEncoder">
            <pattern>${CONSOLE_LOG_PATTERN}</pattern>
        </encoder>
        <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
            <!-- 日志文件命名，按小时切割 -->
            <fileNamePattern>${LOG_HOME}/project-name-%d{yyyy-MM-dd-HH}.log</fileNamePattern>
            <!-- 保留日志文件的天数 -->
            <maxHistory>180</maxHistory>
        </rollingPolicy>
        <!-- 文件大小达到 60MB 时触发切割 -->
        <triggeringPolicy class="ch.qos.logback.core.rolling.SizeBasedTriggeringPolicy">
            <maxFileSize>60MB</maxFileSize>
        </triggeringPolicy>
    </appender>

    <!-- Spring 框架日志级别配置 -->
    <logger name="org.springframework" level="info" />

    <!-- Hibernate 日志配置 -->
    <logger name="org.hibernate.type.descriptor.sql.BasicBinder" level="TRACE" />
    <logger name="org.hibernate.type.descriptor.sql.BasicExtractor" level="DEBUG" />
    <logger name="org.hibernate.SQL" level="DEBUG" />
    <logger name="org.hibernate.engine.QueryParameters" level="DEBUG" />
    <logger name="org.hibernate.engine.query.HQLQueryPlan" level="DEBUG" />

    <!-- MyBatis 日志配置 -->
    <logger name="com.apache.ibatis" level="TRACE" />
    <logger name="java.sql.Connection" level="DEBUG" />
    <logger name="java.sql.Statement" level="DEBUG" />
    <logger name="java.sql.PreparedStatement" level="DEBUG" />

    <!-- 全局日志输出级别与输出源 -->
    <root level="info">
        <appender-ref ref="STDOUT" />
        <appender-ref ref="FILE" />
    </root>
</configuration>
```

### 配置说明：

1. **日志文件存储**：`LOG_HOME` 属性指定了日志文件的存储目录，建议使用绝对路径，避免相对路径带来的不确定性。
2. **日志格式**：通过 `CONSOLE_LOG_PATTERN` 属性自定义日志输出格式，包括时间戳、线程信息、日志级别、日志消息等。

3. **日志输出**：日志通过两个输出端口：

   - `STDOUT`：将日志输出到控制台。
   - `FILE`：将日志按小时分割存储到文件中，最多保留 180 天的日志文件，每个文件大小不超过 60MB。

4. **特定框架的日志级别**：为 Spring、Hibernate、MyBatis 等框架设置了单独的日志级别，方便调试和问题排查。

5. **全局日志级别**：设置全局日志级别为 `info`，可根据实际需求调整为 `debug` 或 `error`。

通过以上配置，你可以灵活地控制 `tio-boot` 应用中的日志记录行为，确保日志输出简洁、规范，便于后期分析和问题定位。
