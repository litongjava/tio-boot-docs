# api 管理

本项目的目标是构建一个 RAG（Retrieval-Augmented Generation）系统。以下是系统中的核心概念：

- **应用**：知识库的集合，可为每个应用自定义提示词，以满足个性化需求。
- **知识库**：由多个文档组成，便于用户对内容进行分类和管理。
- **文档**：系统中对应的真实文档。
- **片段**：文档经过拆分后的最小内容单元，用于更高效的处理和检索。

## 已完成的功能

在前面的开发中，我们已经完成了以下模块：

1. **用户登录**

   - 实现了安全的用户认证系统，保障用户数据的安全性，并限制未经授权的访问。

2. **模型管理**

   - 支持对不同平台的模型进行管理，包括 OpenAI、Google Gemini 和 Claude。

3. **知识库管理**

   - 支持创建、更新和删除知识库，方便用户高效地维护和管理内容。

4. **文档拆分**

   - 提供将文档拆分为多个片段的功能，为后续向量化和检索做好准备。

5. **片段向量化**

   - 将文本片段进行向量化处理，便于语义相似度计算和高效检索。

6. **命中率测试**

   - 基于用户问题，利用语义相似度和 Top-N 算法进行向量检索，返回与问题最相关的片段。

7. **文档管理**

   - 提供上传、管理知识库中的文档功能，并支持文档自动拆分为片段。

8. **片段管理**

   - 支持对拆分后的片段进行增删改查操作，确保内容灵活可控。

9. **问题管理**

   - 支持为片段指定相关问题，提高检索的准确性。

10. **应用管理**

    - 支持创建和配置应用（智能体），并关联特定的模型和知识库。

11. **向量检索**

    - 基于语义相似度算法，从知识库中检索与用户问题相关的片段。

12. **推理问答调试**

    - 评估系统的检索和问答性能，并提供优化系统效果的工具。

13. **对话问答**

    - 提供友好的交互界面，基于检索到的片段和用户的问题实时生成回答。

14. **统计分析**
    - 对用户问答数据进行统计分析，并以图表形式展示，帮助用户理解系统使用情况。
