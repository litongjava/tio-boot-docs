# Discover API 文档

本文档详细介绍了 `/api/discover` 接口的功能、返回数据结构及其实现代码，便于开发人员和维护人员理解和使用该接口。

## 1. 接口说明

- **接口地址**：`GET /api/discover`
- **接口功能**：返回博客或新闻内容的列表，主要用于展示相关内容的摘要信息。

## 2. 返回数据格式

接口返回一个 JSON 对象，主要包含一个 `blogs` 数组，每个数组元素代表一条博客或新闻记录。示例响应如下：

```json
{
  "blogs": [
    {
      "similarity": null,
      "thumbnail": "https://www.bing.com//th?id=OVFT.J2ES8dqSTKgpQ0RBA2V0XC&pid=News&w=234&h=132&c=14&rs=2&qlt=90",
      "description": null,
      "url": "https://www.yahoo.com/tech/chinese-research-team-makes-game-103007466.html",
      "title": "Chinese research team makes game-changing breakthrough with floating solar tech: 'Our work made significant progress",
      "content": "Our work made significant progress in understanding the hydrodynamic responses of a novel offshore floating photovoltaic ..."
    }
  ]
}
```

### 字段说明

- **similarity**  
  当前记录的相似度信息，目前返回为 `null`，可能在后续扩展中用于表示相关性评分。

- **thumbnail**  
  缩略图链接，提供了图片资源的 URL，方便前端进行图片展示。

- **description**  
  文章摘要或描述，目前返回为 `null`，后续可用于提供更详细的内容简介。

- **url**  
  原始内容的链接地址，通过该链接可以跳转至详细页面或原文。

- **title**  
  标题信息，简明扼要地描述了博客或新闻的主题。

- **content**  
  文章部分内容，用于展示文章内容的预览，帮助用户快速了解内容概要。

## 3. 后端实现代码

以下是接口对应的 Java 代码实现，该代码使用了自定义注解来实现路由映射和请求方法的定义：

```java
package com.litongjava.max.search.controller;

import java.util.ArrayList;
import java.util.List;

import com.litongjava.annotation.Get;
import com.litongjava.annotation.RequestPath;
import com.litongjava.max.search.vo.DiscoverResultVo;
import com.litongjava.model.web.WebPageContent;

@RequestPath("/api/discover")
public class ApiDiscoverController {

  @Get
  public DiscoverResultVo index() {
    List<WebPageContent> blogs = new ArrayList<>();
    WebPageContent webPageContent = new WebPageContent()
        .setUrl("https://www.yahoo.com/tech/chinese-research-team-makes-game-103007466.html")
        .setTitle("Chinese research team makes game-changing breakthrough with floating solar tech: 'Our work made significant progress")
        .setThumbnail("https://www.bing.com//th?id=OVFT.J2ES8dqSTKgpQ0RBA2V0XC&pid=News&w=234&h=132&c=14&rs=2&qlt=90")
        .setContent("Our work made significant progress in understanding the hydrodynamic responses of a novel offshore floating photovoltaic ...");
    blogs.add(webPageContent);
    DiscoverResultVo discoverResultVo = new DiscoverResultVo(blogs);
    return discoverResultVo;
  }
}
```

### 代码解析

- **路由映射**

  - 类上使用 `@RequestPath("/api/discover")` 注解，标识该控制器处理 `/api/discover` 路径下的请求。
  - 方法上使用 `@Get` 注解，表明该方法响应 GET 请求。

- **业务逻辑**
  - 在 `index` 方法中，创建了一个 `List<WebPageContent>` 用于存放返回的博客内容。
  - 构造 `WebPageContent` 对象，并依次设置文章的 URL、标题、缩略图和内容等信息。
  - 将该对象添加至列表后，用列表构造一个 `DiscoverResultVo` 实例，并返回该实例。

## 4. 使用说明

- **调用方式**：客户端通过 `GET` 请求访问 `/api/discover` 即可获取内容列表。
- **扩展方向**：
  - 后续可以通过增加相似度（similarity）、描述（description）等字段来丰富返回信息。
  - 可以通过参数控制返回内容的数量或分页处理，以适应大规模数据返回的场景。

## 5. 总结

该接口通过简洁的代码实现，实现了将博客或新闻内容封装为 JSON 数据返回给客户端。代码结构清晰，扩展性较强，可根据需求进一步完善数据字段及业务逻辑，为前端提供稳定的数据接口支持。
