# manim 片段流式传输方案

## 1. 背景概述

在使用 manim 生成视频与音频过程中，会分别生成视频片段和音频片段，存储在不同目录下（例如 video 与 audio 目录）。为了实现前端的流式播放，需要将这两部分内容转换为 HLS（HTTP Live Streaming）格式，即将文件转换为 TS 分段，并生成对应的 m3u8 播放列表。该方案的关键在于将视频与音频分离管理，并通过主播放列表将两者合并，从而实现音视频同步播放。

## 2. 系统设计思路

### 2.1 分离管理视频与音频

- **视频管理：**  
  将 video 目录下的 manim 生成视频片段转换为 TS 分段，并将分段信息追加到 `playlist_video.m3u8` 播放列表中。

- **音频管理：**  
  将 audio 目录下的 manim 生成音频片段转换为 TS 分段，并将分段信息追加到 `playlist_audio.m3u8` 播放列表中。

这种分离管理方式便于各自独立更新，同时在后续合并时可以更好地处理不同媒体的同步问题。

### 2.2 生成主播放列表（Master Playlist）

为了让播放器同时加载音视频并保持同步，需要生成一个主播放列表。主播放列表通过 `EXT‑X‑MEDIA` 标签分别引用视频和音频播放列表，实现音视频的统一管理与同步播放。  
示例格式如下：

```
#EXTM3U
#EXT-X-VERSION:3
#EXT-X-MEDIA:TYPE=AUDIO,GROUP-ID="audio",NAME="Audio",DEFAULT=YES,URI="playlist_audio.m3u8"
#EXT-X-STREAM-INF:BANDWIDTH=800000,RESOLUTION=640x360,CODECS="avc1.42e01e,mp4a.40.2",AUDIO="audio"
playlist_video.m3u8
```

### 2.3 独立转换与追加

- **转换接口调用：**  
  当 manim 连续生成视频和音频时，可以分别调用类似 `NativeMedia.appendMp4ToHLS` 的接口，将 MP4 文件转换为 HLS 所需的 TS 分段。

- **追加播放列表：**  
  转换后的 TS 分段信息分别追加到 `playlist_video.m3u8` 和 `playlist_audio.m3u8` 中，实现播放列表的实时更新。

### 2.4 同步问题注意事项

由于视频和音频数据存储分离，为保证最终播放效果，必须确保：

- **时间戳一致：** TS 分段的时间信息需要保持一致，避免播放时出现错位。
- **序号同步：** 分段序号必须连续且对应，防止前端播放器在合成时出现音视频不同步的情况。

## 3. 实现流程

1. **视频与音频分段转换：**

   - 每当有新生成的 video 或 audio 文件，调用转换接口（例如 `NativeMedia.appendMp4ToHLS`）将文件转换为 TS 分段。
   - 将转换后的 TS 分段分别追加到 `playlist_video.m3u8` 与 `playlist_audio.m3u8` 中。

2. **播放列表更新：**

   - 随着 TS 分段的追加，确保播放列表文件保持最新状态，供前端播放器实时加载。

3. **主播放列表构建：**

   - 在 `master.m3u8` 中使用 `EXT‑X‑MEDIA` 和 `EXT‑X‑STREAM-INF` 标签引用上述两个播放列表，实现音视频的合并播放。

4. **数据流式返回前端：**
   - 在视频片段的第一批生成后，通过主播放列表 `master.m3u8` 将数据流式返回到前端，确保前端能够尽快启动播放。

## 4. 注意事项与最佳实践

- **同步校准：**  
  建议在转换及追加过程中加入同步校准机制，确保音视频分段在时间戳和序号上的一致性，避免播放中出现卡顿或不同步的情况。

- **实时更新：**  
  播放列表文件需支持实时追加与更新，前端播放器也应具备动态加载最新 TS 分段的能力。

- **错误处理：**  
  在转换或追加过程中需考虑异常处理机制，保证播放列表数据的完整性，防止因某一分段转换失败而影响整体播放体验。
